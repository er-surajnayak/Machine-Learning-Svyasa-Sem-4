Data Processing
Methods

Preprocessing is where 70% of ML intelligence happens. Three techniques — discretization, normalization, and one-hot encoding — transform raw messy data into model-ready numbers.

01 · The overview

Why preprocessing matters

Real-world data is never clean. You'll see missing values, text where numbers are expected, features on wildly different scales, and categories that need special handling. The model can't learn from raw chaos.

Bad preprocessing = bad model. No matter how sophisticated your algorithm is, if the input data is poorly prepared, the model learns the wrong patterns. Garbage in, garbage out.
Three preprocessing pillars
DiscretizationBreak continuous values into categories
NormalizationBring features to the same scale
One-Hot EncodingConvert categories to binary columns
02 · Method 1

Discretization — grouping continuous values

Discretization converts continuous numerical values into discrete categories (bins). Instead of exact numbers, you work with ranges.

Analogy — Marks to grades: Scores 32, 47, 76, 89, 58 become Fail, Average, Good, Excellent. The exact marks disappear — only the group matters. That's discretization.
Example: converting exam marks to grades
Raw markBin rangeCategory
320–40Fail
5841–60Average
7661–80Good
8981–100Excellent
pythonpd.cut
import pandas as pd

df["Grade"] = pd.cut(df["Marks"],
                     bins=[0, 40, 60, 80, 100],
                     labels=["Fail", "Average", "Good", "Excellent"])
Trade-off: Discretization reduces noise and simplifies patterns — but you lose detailed information. Someone scoring 60 and 41 both become "Average." Precision is reduced. Use when categories are more meaningful than exact values.
03 · Binning types

Four types of discretization

Equal Width Binning

Same range per bin

Divide the entire range into equal-sized intervals. Width = (Max - Min) / n_bins.

Range 0–100, 5 bins:
0–20 / 21–40 / 41–60 / 61–80 / 81–100

⚠ Ignores data distribution — bins can be very unequal in terms of data points.

Equal Frequency Binning

Same count per bin

Each bin has roughly the same number of data points. Looks at distribution.

100 students, 4 bins:
Bottom 25% / Next 25% / Next 25% / Top 25%

✓ Balanced groups. Good for ranking problems.

Domain-Based

Expert-defined bins

Bins defined using domain knowledge, not math.

Blood pressure:
<90=Low / 90–120=Normal / 121–139=Elevated / 140+=High

✓ Medically/scientifically meaningful. Used in healthcare, finance.

Model-Based

Algorithm finds optimal splits

Decision trees automatically find best split points from data.

IF Marks > 52.5 → Pass
ELSE → Risk

✓ Data-driven. Model discovers the optimal threshold.

pythonequal width vs frequency
# Equal Width — pd.cut with integer bins
df["Grade"] = pd.cut(df["Marks"], bins=4, labels=["Low", "Mid", "High", "Top"])

# Equal Frequency — pd.qcut
df["Grade"] = pd.qcut(df["Marks"], q=4, labels=["Low", "Mid", "High", "Top"])
04 · Method 2

Normalization — bringing features to the same scale

When features have very different ranges, the larger-scale feature dominates the model — not because it's more important, but because its numbers are bigger. Normalization fixes this.

The basketball analogy: If one player suddenly becomes 60 feet tall, the game is unfair. Normalization makes sure everyone plays on the same scale.

Min-Max Normalization

The most common technique. Rescales all values to the range 0 to 1.

Xnorm = (X − Xmin) / (Xmax − Xmin)
Result always between 0.0 and 1.0

scaler = MinMaxScaler()
df["CGPA"] = scaler.fit_transform(df[["CGPA"]])
# All CGPA values now between 0.0 and 1.0
05 · Normalization type 2

Z-Score Standardization

Instead of squeezing into 0–1, Z-score centers data around zero. It measures how far each value is from the average, expressed in standard deviations.

Z = (X − μ) / σ
μ = mean, σ = standard deviation · Result: mean=0, std=1
Analogy: Instead of saying "you scored 80," say "you scored 1.5 standard deviations above average." Now you understand performance relative to the class — not just as an absolute number.

06 · Method 3

One-Hot Encoding

Categorical data can't be directly fed to most ML models. If we encode CSE=1, ECE=2, MECH=3, the model wrongly assumes MECH > CSE. One-hot encoding solves this by creating separate binary columns.

The idea: Instead of one "Branch" column with values 1/2/3, we create three columns — one for each category — with binary 0 or 1.

pythonpd.get_dummies
import pandas as pd

df = pd.DataFrame({"Branch": ["CSE", "ECE", "MECH", "CSE"]})
df_encoded = pd.get_dummies(df, columns=["Branch"])

# Creates: Branch_CSE, Branch_ECE, Branch_MECH
# Each column is 0 or 1 — binary checkbox
Note on dimensionality: If you have 100 unique categories, one-hot encoding creates 100 new columns. That's a big increase in dimensions. For large categorical features, consider other encoding methods (label encoding for ordinal, target encoding, etc.)

08 · Check your understanding

Quick quiz

1. CGPA ranges from 0–10, Salary ranges from 0–10,00,000. Which preprocessing step is needed to prevent salary from dominating?
A. One-hot encoding
B. Discretization
C. Normalization ✓
D. No preprocessing needed
Normalization (Min-Max or Z-Score) brings both features to the same scale. Without it, the model gives more weight to salary simply because its numbers are larger — not because it's more important. ✓
2. You have a "City" column with 50 unique cities. After one-hot encoding, how many new columns are created?
A. 1 column
B. 10 columns
C. 50 columns ✓
D. 2 columns
One-hot encoding creates one binary column per unique category. 50 cities = 50 new columns. This is why one-hot encoding on high-cardinality features can explode the feature space! ✓
3. Z-Score standardization transforms data so that which of these is true?
A. All values are between 0 and 1
B. Mean = 0, Standard deviation = 1 ✓
C. All values become positive
D. Minimum becomes 0, Maximum becomes 100
Z-Score standardization centers data at mean=0 and std=1. Values above the mean become positive, below become negative. This is different from Min-Max which always gives a 0–1 range. ✓
