Introduction to
Machine Learning

From apples and oranges to cancer detection — understanding how machines learn from examples instead of rules, and the four fundamental types of learning.

01 · The concept

What is Machine Learning?

Let's start with something simple. Imagine you're teaching a kid how to recognize apples and oranges. You don't give a definition. You just show examples — "This is apple. This is orange." After seeing enough examples, the kid starts recognizing patterns.

That's Machine Learning. Teaching a computer using examples instead of writing strict rules.

Traditional programming vs ML: In traditional programming you write: "If marks > 40 → Pass." But what about someone with 7.5 CGPA, 3 internships, and strong projects? Instead of complicated rules, we give the computer past data — and let it find the patterns itself.
Traditional Programming
ApproachYou write the rules
InputRules + Data
OutputAnswers
LimitationCan't handle complexity
Machine Learning
ApproachSystem learns the rules
InputData + Answers
OutputRules (learned patterns)
StrengthHandles messy real-world data

The one-liner: Machine Learning = Finding patterns from data.

02 · Type 1

Supervised Learning

This is the most common type. The model learns from labeled data — for every input, we already know the correct output. It learns by comparing its prediction with the real answer and correcting itself.

Analogy: It's like solving previous year question papers where the answer key is given. You learn by comparing your answer with the correct one and adjusting.

03 · Type 2

Unsupervised Learning

No labels. No correct outputs. Just raw data. The model must find patterns on its own.

Analogy: On the first day of college, nobody assigns friend groups. Still — coders sit together, cricket fans sit together, same hometown students sit together. No label was given. Groups formed naturally. That's unsupervised learning.
Example — Customer segmentation

Only age and spending score — no "Premium Customer" label given. The model groups similar customers automatically.

Real-world examples: Spotify grouping music taste, news apps grouping similar articles, market analysis. Unsupervised learning = discovering hidden structure.

04 · Type 3

Semi-Supervised Learning

A mix of both. Some data is labeled. Most is not. Very common in real life because labeling data is expensive.

The solution

ResultBetter accuracy
SavingsHuge reduction in cost
Simple analogy: You solve 10 math problems with answers. Then you attempt 40 similar ones without answers. You use earlier understanding to solve new ones. That's semi-supervised learning.
05 · Type 4

Reinforcement Learning

Completely different. No dataset. No correct answers. Instead: an agent interacts with an environment and receives rewards or penalties.

Agent → Takes Action → Gets Reward or Penalty → Learns Strategy → Repeat
Analogy — Learning to ride a bike: First attempt → fall (penalty). Second → better balance. After many tries → smooth ride. Nobody gave step-by-step instructions. You learned through feedback.
Self-driving cars

Stays in lane → reward. Crashes → heavy penalty. Learns optimal driving over millions of simulations.

Gaming AI

Wins → +reward. Loses → penalty. Improves strategy to maximize reward over time.

Robotics

Robot walks, falls, adjusts. Reinforcement signal guides it toward stable movement.

06 · Side by side

Quick comparison — all four types

Click each type to expand the details.

S
Supervised Learning
›
Data: Labeled (input + correct output)
Goal: Learn a mapping from input to output
Key insight: Learns from answers — like studying with an answer key
Used for: Classification (spam, pass/fail), Regression (price, salary)
Examples: Email spam, house price prediction, disease diagnosis
U
Unsupervised Learning
›
Data: Unlabeled (no output column)
Goal: Find hidden structure or patterns
Key insight: Discovers groups naturally — like friend circles forming on day 1
Used for: Clustering, dimensionality reduction, anomaly detection
Examples: Customer segmentation, topic modelling, image compression
SS
Semi-Supervised Learning
›
Data: Small labeled + large unlabeled
Goal: Use few labels to guide learning on many unlabeled examples
Key insight: A little guidance goes a long way — like solving practice questions then applying logic to new ones
Used for: Medical imaging, text classification with limited annotations
Examples: X-ray classification, Google Photos face recognition
RL
Reinforcement Learning
›
Data: No dataset — learns from environment interaction
Goal: Maximize cumulative reward over time
Key insight: Learn by doing and getting feedback — just like riding a bike
Used for: Robotics, game AI, self-driving cars, trading bots
Examples: AlphaGo, Tesla Autopilot, Amazon warehouse robots

07 · Real world

Applications of ML everywhere

ML is not one algorithm. It's pattern recognition applied across every domain. If there is data, ML can potentially learn from it.

Social Media

Instagram & YouTube

You watch 3 gym reels → your feed fills with fitness content. The system learns from what you watch, how long, what you skip.

Banking

Fraud Detection

Card used in another country → transaction declined instantly. The model learned your spending pattern and flagged the deviation.

Healthcare

Disease Prediction

ML detects tumors in X-rays, predicts diabetes risk. It assists doctors — it doesn't replace them.

Entertainment

Netflix

100 people watch the same movie. Their homepages look different. The system predicts what each person will enjoy next.

Transport

Uber Dynamic Pricing

Price shoots up when it rains. The system considers demand, drivers available, time, weather, and location — all in real time.

Industry

Predictive Maintenance

A machine vibrates differently → "This machine might fail in 3 days." Factories save millions by predicting failures early.
